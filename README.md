# Notes Management App

A simple Flutter notes app backed by Cloud Firestore. Notes are created, listed,
edited, and deleted against a live Firestore database, and the list updates in
real time as the data changes.

## Features

- **Create** — add a note with a title and description
- **View** — all notes listed newest-first, streamed live from Firestore
- **Update** — edit an existing note's title or description
- **Delete** — remove a note from the database

## Screens

| Screen | File | Purpose |
| --- | --- | --- |
| Notes List | `lib/screens/notes_list_screen.dart` | Shows all notes; tap to edit, trash icon to delete, FAB to add |
| Add / Edit Note | `lib/screens/note_edit_screen.dart` | One form serving both create and edit modes |

## Data model

A single Firestore collection, `notes`. Each document:

| Field | Type | Notes |
| --- | --- | --- |
| `title` | string | Required, non-empty |
| `description` | string | Required, non-empty |
| `createdAt` | timestamp | Set on create via `serverTimestamp()`; orders the list |

## Project structure

```
lib/
├── main.dart                     # Firebase init + MaterialApp
├── firebase_options.dart         # Generated by `flutterfire configure`
├── models/note.dart              # Note model + Firestore mapping
├── services/notes_service.dart   # All Firestore reads/writes
└── screens/
    ├── notes_list_screen.dart
    └── note_edit_screen.dart
```

State is handled with a plain `StreamBuilder` over the Firestore snapshot
stream — no state-management package. Firestore pushes changes, the list
rebuilds itself, and nothing needs a manual refresh.

## Running it

This repo's `firebase_options.dart` and `google-services.json` point at the
original author's Firebase project. To run it against your own:

1. Install the tooling:
   ```sh
   npm install -g firebase-tools
   dart pub global activate flutterfire_cli
   firebase login
   ```
2. In the [Firebase console](https://console.firebase.google.com), create a
   project, then **Build → Firestore Database → Create database → test mode**.
3. From the project root, point the app at your project:
   ```sh
   flutterfire configure --platforms=android
   ```
4. Install dependencies and run:
   ```sh
   flutter pub get
   flutter run
   ```

Firestore runs in **test mode** (open read/write, no authentication), which is
appropriate for this assignment but not for production. Test-mode rules expire
30 days after creation.
